# Настройка интеграции

Настройка интеграции доступна после передачи токена доступа Yandex.Market в личном кабинете СДЭК.
Подробнее об этом можно прочитать на [отдельной странице](./yandex-key.mdx).

<Image img={require('./img/setup/init.png')} />

:::danger

Если вы **удаляете интеграцию** после настройки, или **отзываете токен доступа в личном кабинете Yandex.Market**, или **изменяете в личном кабинете СДЭК токен на некорректный**, то тем самым вы даете согласие на удаление всех данных, связанных с интеграцией, в том числе, и созданных накладных.

Если при этом часть накладных будут в пути, то они будут возвращены отправителю **с выставлением счета на оплату за
полную стоимость доставки**.

:::


### Данные о магазине {#shop}

Основные настройки задаются в разделе настроек магазина.

<details>
    <summary>Секция данных о магазине</summary>

    <Image img={require('./img/setup/shop/main.png')} />

    Данные из поля названия будут использоваться в качестве продавца в накладной.

</details>

<details>
    <summary>Секция тарифов</summary>

    <Image img={require('./img/setup/shop/tariff.png')} />

    Данные из полей секции будут участвовать в алгоритме определения тарифа, с которым будет создана накладная.

    :::warning

    При выборе тарифов рекомендуется выбирать как можно больше вариантов, включая тариф `Посылка` и `Экспресс`.
    Это позволит сократить количество необработанных накладных.

    :::

    Подробнее про алгоритм подбора тарифов можно прочитать в [отдельной статье](./creation.mdx#create).

    <details>
        <summary>Нельзя отключить `Дополнительный сбор за объявленную стоимость`</summary>

        Данная услуга является обязательной для Интернет-Магазинов, поэтому отключить ее нельзя.
        Стоимость товара подтягивается из карточки товара Yandex.Market. Дополнительный сбор рассчитывается
        автоматически на основе стоимости товара.

        Подробнее можно прочитать в [Регламенте возмездного оказания курьерских услуг](https://www.cdek.ru/ru/oferta/).

    </details>

</details>

<details>
    <summary>Секция товаров</summary>

    <Image img={require('./img/setup/shop/goods.png')} />

    :::info

    Данная секция не является обязательной к заполнению.

    :::

    Если у карточки товара в Yandex.Market не указан параметр, то будет использоваться значение из поля в данной
    секции для вычисления стоимости доставки.

</details>

<details>
    <summary>Секция уведомлений</summary>

    <Image img={require('./img/setup/shop/notifications.png')} />

    :::info

    Данная секция не является обязательной к заполнению.

    :::

    Настройки в данной секции позволяют оперативно получать уведомления, связанные с интеграцией.

    - Об ошибках: каждый раз при попытке создания накладной, если возникла ошибка, будет отправлено сообщение
    - О заказах: в соответствии с выбранными настройками, будет отправлен список обработанных заказов и созданных к
    ним накладных
    - Новости: при добавлении новых функций или изменении работы интеграции, будет отправлено сообщение с описанием изменений

    :::note

    В настоящий момент работает только рассылка уведомлений об ошибках на почту.

    :::
</details>

:::note

После нажатия на кнопку **Сохранить**, настройки магазина будут применены в течение 5 минут.

:::

### Данные о складе

Для каждого склада задаются условия отправки груза и данные для накладной.

Если вы не создавали склады в Yandex.Market, то сделайте это по [инструкции](https://yandex.ru/support/marketplace/ru/storage/storage-fbs-dbs).

:::note

Склады и их наименование загружаются из Yandex.Market автоматически раз в сутки.

Для обновления данных вручную, необходимо нажать на кнопку **Синхронизировать склады**.

:::warning

Процесс ручной синхронизации может занимать продолжительное время в зависимости от количества складов и текущей
нагрузки на систему.

:::

:::tip

Вы можете самостоятельно настроить плату за доставку, которую Yandex.Market будет взимать с покупателя, по
[инструкции](https://yandex.ru/support/marketplace/ru/orders/dbs/tariffs).

:::

После нажатия на кнопку **Сохранить**, настройки склада будут применены в течение 5 минут.

### Активация магазина

Для активации магазина, необходимо нажать на кнопку **Активировать**.

<Image img={require('./img/setup/activate.png')} />

:::warning

Активация интеграции станет доступна только после полной настройки магазина и хотя-бы одного склада.

:::

:::danger

**С момента активации магазина должно пройти не менее 30 минут для корректной работы интеграции.**
Это связано с особенностями работы API Yandex.Market.

Любые действия со сборочными заданиями до этого времени могут привести к некорректной работе интеграции и накладные по
таким заданиям созданы не будут.

:::

<details>
    <summary>Деактивация магазина</summary>

    Для деактивации магазина, необходимо нажать на кнопку **Деактивировать**.

    <Image img={require('./img/setup/deactivate.png')} />

    :::note

    Магазин деактивируется в течение 5 минут.

    :::

</details>

После активация магазина, интеграция начнет загружать сборочные задания из Yandex.Market и можно будет работать с
накладными.
